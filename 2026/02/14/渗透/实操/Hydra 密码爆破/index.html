Hydra密码爆破

Hydra密码爆破测试

原本跟着教程想试一下破解电脑系统密码，到最后因为版本问题失败了，这里主要做复盘（以下所有操作均在Kali Linux中完成）

信息收集阶段

渗透测试首先要做好信息收集工作，在此任务进程中主要用到的指令有ifconfig (网卡及本地IP嗅探)，netdiscover（同局域网下其他设备），nmap(对特定设备进行扫描获取信息)

  1. 首先确定IP地址所在

    在命令行中输入ifconfig指令，获取本地信息

    得知本地ip为192.168.201.129

  2. 再扫描本地局域网IP下的其他设备

    在命令行中输入netdiscover 指令，对其他设备进行扫描嗅探

    这里扫描到我的另一台虚拟机，虚拟机系统Windows Server 2022,我想要以此虚拟机作为攻击对象进行训练

  3. 确定好目标后对目标进行网络扫描

    在命令行中输入nmap 指令，对目标进行可攻击端口扫描

    nmap 192.168.201.131

    获取到以下信息

    通过AI了解到各端口链接的功能

    根据各协议，决定选择445端口SMB协议进行hydra 爆破

密码攻击阶段

使用kali linux 自带的Hydra 工具进行密码爆破，但是Hydra爆破需要使用密码字典，所以这里还需要使用Kali Linux自带的另一个工具wordlists

工具使用规则：

hydra -l user -P passlist.txt ftp://192.168.0.1

  1. 使用wordlists 生成密码字典

    在菜单栏搜索该程序或者新建任务窗口输入wordlists 指令

  2. 利用每个计算机都有的账户administrator作为爆破账户，选择rockyou.txt作为密码字典进行爆破

    hydra -l administrator -P usr/share/wordlists/rockyou.txt 192.168.201.130 SMB

正常情况下到这里就差不多爆破成功了，但是我在运行上述指令过程中出现报错，问了AI后了解到攻击目标的系统版本对上述简单的Hydra 爆破有阻断

AI解释：

目标端口开着，但 SMB 协议版本 / 签名要求太高
• Windows 10/11、Server 2016+ 默认强制 SMB 3.1.1 + 签名，Hydra 的 smb 模块还停留在 NT LM 0.12，握手阶段就被拒绝。
• 某些 NAS、新版本 Samba 4.11+ 也默认禁用 SMB1。

总结：

  1. Hydra密码爆破一般只用于特定的目标版本
  2. Hydra 的 smb 模块只懂古老的 SMB1/NT LM 0.12；只要目标拒绝 SMB1 或返回 SMB2+ 握手，Hydra 就报 “invalid reply”。
    先拿 nmap/crackmapexec 确认服务与协议版本，如果目标确实只接受 SMB2/3，就换工具，别在 Hydra 上继续浪费时间
  3. Hydra的密码爆破不是万能的，依靠的密码本量大，爆破时间长（如果知道目标的一些具体信息，可以使用工具生成针对特定目标的社工密码本，依靠社工密码本，本条问题可以忽略不计）
