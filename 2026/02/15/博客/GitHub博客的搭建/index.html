GitHub博客的重建

刚开始的时候没有好好研究博客的搭建，弄得乱七八糟，然后就搁置了好久，因为最近写了好多笔记，就打算重新收拾一下我的博客，一切推翻重来吧。

仓库的删除

清理GitHub杂乱的内容然后重新开始，通常有两种思路：

一是彻底删除整个仓库后重建（干净利落）

二是只清空仓库内容但保留仓库名和配置（省去重新创建的步骤）

在这里我选择了方案一，正所谓不破不立，破而后立，干脆全删了重新开始吧。如果还想要原来的仓库名，那直接重新设为原来的仓库名得了。

主要有以下几步操作：

  1. 备份（重要）：如果本地还有需要的代码，先做好备份。

  2. 进入设置：在 GitHub 上进入目标仓库，点击顶部导航栏的 Settings 。

  3. 危险区域：在设置页面左侧或底部找到 “Danger Zone”（危险区域）。

  4. 执行删除：点击 “Delete this repository” 按钮。

  5. 确认身份：在弹出的窗口里，按照提示输入 “用户名/仓库名” 来确认，然后点击下方的删除按钮即可

​ 注意：这是一个不可逆操作。删除后代码和所有关联信息将永久消失。不过 GitHub 在删除后 90 天内提供恢复 入口，可以在 Settings -> Repositories -> Deleted repositories 中找回

创建新仓库

  1. 在 GitHub 首页点击 New repository。
  2. 仓库命名：如果你打算用 GitHub Pages 托管，建议直接命名为 用户名.github.io（例如 lisi.github.io），这样 Pages 会自动启用，且访问域名就是 https://用户名.github.io。如果只是普通仓库，可随意命名。
  3. 选择公开（Public）或私有（Private），但 Pages 必须公开才能免费访问。
  4. 可以勾选“Add a README file”方便快速克隆。
  5. 点击 Create repository。

仓库创建完毕，接下来要做的就是初始化本地然后选择博客框架

检查当前连接的远程仓库地址

如果是连接的确定是新建的仓库地址，你会看到如下内容

  • origin：这是远程仓库的默认名称，你可以把它理解成“远程仓库在本地的代号”。
  • https://github.com/...：这就是当前绑定的仓库的真实地址。看到这个地址，你就知道本地仓库正在和哪个 GitHub 仓库通信。
  • (fetch) 和 (push)：分别表示拉取和推送使用的地址。通常两个地址是一样的，表示从同一个地方拉取和推送。

Q2:笔记中图片无法显示

我在生成静态文件然后预览的过程中发现我的图片无法显示，这个时候我才想起我每个文件中的图片地址都是使用的绝对地址

  • 使用 /images/xxx.jpg 是绝对路径（从网站根目录开始），无论文章在哪层子文件夹都能正确引用。
  • 使用 ./xxx.jpg 或 xxx.jpg 是相对路径，依赖于当前文件位置，容易出错。除非使用资产文件夹，否则不建议。

使用绝对地址的话，我上传到远端仓库后或者是我以后整理图片的时候极容易造成地址链接失效的问题，然后我通过AI获取到了使用直链的方式

GitHub + jsDelivr（稳定，需 GitHub 账号）

  • 原理：将图片上传到 GitHub 仓库，通过 jsDelivr CDN 加速访问。
  • 步骤：
    1. 在 GitHub 上新建一个公共仓库（比如 images）。
    2. 克隆仓库到本地或直接在网页上传图片。
    3. 获取图片的直链：https://cdn.jsdelivr.net/gh/你的用户名/仓库名@分支名/图片路径
      • 例如：https://cdn.jsdelivr.net/gh/AZHU-6315/images@main/photo.jpg
    4. 在 Markdown 中引用。

链接解释：

  • GitHub 用户名：xxx
  • 仓库名：images（你新建的图床仓库）
  • 分支：main
  • 图片路径：2025/photo.png

Q3:预览中旁侧目录显示（no title）

Markdown 文件的 Front-matter 中没有正确设置 title 字段，或者格式有误。

我使用的Typora编写的笔记，可能因为设置原因默认无front-matter,所以我的markdown文件都需要进行格式修改，即在笔记开头位置编写yaml段落

用编辑器打开 source/_posts/ 下对应的 Markdown 文件，确保文件开头有以下格式的内容：

  • --- 必须是文件的最开头，不能有空格或空行。
  • 冒号后面必须有一个空格（例如 title: 我的文章）。
  • 如果 title 缺失或写错（如 titile:），Hexo 就无法识别，从而显示 “no title”。
