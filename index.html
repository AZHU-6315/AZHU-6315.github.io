刚开始的时候没有好好研究博客的搭建，弄得乱七八糟，然后就搁置了好久，因为最近写了好多笔记，就打算重新收拾一下我的博客，一切推翻重来吧。

仓库的删除

清理GitHub杂乱的内容然后重新开始，通常有两种思路：

一是彻底删除整个仓库后重建（干净利落）

二是只清空仓库内容但保留仓库名和配置（省去重新创建的步骤）

在这里我选择了方案一，正所谓不破不立，破而后立，干脆全删了重新开始吧。如果还想要原来的仓库名，那直接重新设为原来的仓库名得了。

主要有以下几步操作：

  1. 备份（重要）：如果本地还有需要的代码，先做好备份。

  2. 进入设置：在 GitHub 上进入目标仓库，点击顶部导航栏的 Settings 。

  3. 危险区域：在设置页面左侧或底部找到 “Danger Zone”（危险区域）。

  4. 执行删除：点击 “Delete this repository” 按钮。

  5. 确认身份：在弹出的窗口里，按照提示输入 “用户名/仓库名” 来确认，然后点击下方的删除按钮即可

​ 注意：这是一个不可逆操作。删除后代码和所有关联信息将永久消失。不过 GitHub 在删除后 90 天内提供恢复 入口，可以在 Settings -> Repositories -> Deleted repositories 中找回

创建新仓库

  1. 在 GitHub 首页点击 New repository。
  2. 仓库命名：如果你打算用 GitHub Pages 托管，建议直接命名为 用户名.github.io（例如 lisi.github.io），这样 Pages 会自动启用，且访问域名就是 https://用户名.github.io。如果只是普通仓库，可随意命名。
  3. 选择公开（Public）或私有（Private），但 Pages 必须公开才能免费访问。
  4. 可以勾选“Add a README file”方便快速克隆。
  5. 点击 Create repository。

虚拟币基础与溯源

课前预习

虚拟币是基于区块链技术的去中心化数字资产，以比特币（BTC）为代表，通过密码学确保交易安全和控制供应。溯源基于区块链的公开透明特性，通过交易哈希（TxHash）、区块链浏览器及专用的数据分析工具追踪资金流向和地址关联。

一、 虚拟币基础

定义与性质：

虚拟币（加密货币）是互联网设计的去中心化数字货币，不依赖中央银行，依靠密码学控制制造和管理。

核心技术：

区块链（分布式账本）技术是其核心，所有交易记录公开且不可篡改。

主流币种：

包括比特币（BTC）、以太坊（ETH）、泰达币（USDT）、索拉纳（SOL）等。

运行机制：

主要通过“挖矿”（如POW机制）获得新币，比特币总量上限为2100万枚。

存储与交易：

用户通过加密钱包（冷钱包/热钱包）存储，在加密货币交易所进行交易

二、 虚拟币溯源技术与方法

由于区块链上的数据是公开的，通过分析链上数据可以追踪资金流向。

  1. 区块链浏览器分析：输入交易ID（TxID）或钱包地址，可查看实时交易流向、时间、金额、地址关联等信息。
  2. 专用溯源设备/系统：公安等机构使用专用分析工具，将地址标记为交易所、热钱包、冷钱包、已知诈骗地址等，生成资金流追溯路径图。
  3. 交易节点追踪：通过追踪交易首次进入比特币网络的服务器节点，可以缩小范围查找真实的客户端节点。
  4. 技术局限：对混币器、匿名币（如门罗币XMR）及跨链协议等高级匿名手段的追踪存在较高技术挑战

正课部分

区块链扫盲

概念

区块链是一种分布式、去中心化的网络数据系统。

特性

唯一性

数据的唯一历史：通过密码学和共识机制，创造了一条不可篡改、唯一可信的数据序列。

数字产权的唯一标识：为数字或实物资产提供了一个不可复制、可验证的唯一所有权证明。

状态的一致性：在全球分散的节点间，就一个唯一的真相达成一致。

去中心化

用通俗一点的话来讲就是“众生平等”，在所有交易对象中没有人有特殊权限，除了你没人能够验证你的身份，多个节点共同协同存储数据，每有一个新人想要做业务，便会通过共识机制，广播到所有节点的用户，确保每个节点具有相同的数据存储内容。

共识机制

它是整个去中心化网络在没有中央权威的情况下，就“哪个数据是真实的”以及“如何按顺序记录数据”达成一致的一套规则。简单来说，就是在去中心化的环境下，去处理记帐权、交易真实性和新区块打包的问题。

打个比方：
想象一个没有村长、也没有会计的村子，所有村民自己记账。

问题：张三说自己给了李四10块钱，李四说只收到5块，该信谁？怎么保证没人偷偷改自己的账本？

解决方案（共识机制）：

规则：全村约定，每天下午大家聚在一起，比赛做一道很难的数学题。第一个做出题的人，获得大声宣布“今天所有交易”的权利。

核对：其他村民核对这份宣布的交易是否合理（比如，花钱的人有没有足够的钱）。

确认：如果大多数人都说“没问题”，那么全村人就统一把这笔交易记到各自的账本新的一页上，并以这页为基础开始下一页。

这个“比赛做题、一起核对、统一记账”的规则，就是共识机制。

防篡改

区块链通过哈希算法进行链式储存，任何人在做完一个转账后，账本会计算出一个哈希值，再进行转账操作会验证前一步的哈希值，基本无法修改哈希值来篡改账本。

可追溯

可以通过交易哈希和钱包地址等进行交易追溯

匿名性

公链上只保存用户的钱包地址，不保存个人信息，类似银行卡号，但区块链上一个人是可以拥有很多钱包地址的。私有链、联盟链不需要匿名性

类型

公有链

一个完全去中心化、对全世界开放的区块链网络。任何人，在任何地点，只要下载了对应的软件，都可以成为网络中的一个节点，参与交易、验证和记账。代表：比特币，以太坊，Solana等

私有链

一个完全去中心化、对全世界开放的区块链网络。任何人，在任何地点，只要下载了对应的软件，都可以成为网络中的一个节点，参与交易、验证和记账。

联盟链

由多个预先选定的、彼此有业务往来的组织共同管理和使用的区块链网络。网络由这些成员组成的联盟共同维护，权限管理是部分去中心化的。

常见名词术语

UTXO

未消费交易输出，比特币特有支付模型。交付制不是账户数值扣除，而是更像现金支付，把钱给别人后，让别人找零。

常见共识机制
工作量证明（PoW）

“谁干活多，谁说了算”

代表：比特币、早期的以太坊。

如何工作：节点（矿工）通过进行大量的哈希计算，来竞争解决一个极其困难的数学难题。第一个解出难题的矿工，获得打包新区块的权利和奖励。

核心：消耗真实的物理资源（电力、算力）。

优点：

  • 安全性极高：要攻击网络需要掌握超过**51%**的全网算力，成本巨大。
  • 高度去中心化（理想情况下）。

缺点：

  • 极其耗能，不环保。
  • 性能低下（交易速度慢，吞吐量低）。
  • 容易导致算力集中，形成“矿池”中心化。
权益证明（PoS）

“谁持币多且愿意锁仓，谁说了算”
代表：以太坊2.0、Cardano、Solana等。

如何工作：验证者需要质押一定数量的原生代币作为“保证金”。系统通过算法（考虑质押数量、质押时间等因素）随机选择一名验证者来打包下一个区块。作恶者的质押金将被罚没。

核心：消耗经济资源（质押的资产）。

优点：

  • 能源效率高，节能环保。
  • 性能通常比PoW更高。

对称加密

就像用一个钥匙和一把锁。加密和解密用同一把钥匙。你必须把钥匙安全地交给对方，对方才能开锁。

常见算法：AES（高级加密标准，目前最常用）、DES、3DES。

总结：对称加密是“锁和钥匙”模式，高效但密钥交换是痛点

非对称加密

就像用一个信箱系统。你有两种钥匙：

公钥：就像信箱的投递口。任何人都可以拿到，用来加密信息或验证签名。

私钥：就像信箱的取件钥匙。只有你一人持有，用来解密信息或进行签名

常见算法：RSA（最经典）、ECC（椭圆曲线加密，比特币/以太坊使用，密钥更短，安全性更高）

密钥关系：

用公钥加密的内容，只能用对应的私钥解密。

用私钥签名的内容，可以用对应的公钥验证其真伪和完整性。

两大核心应用场景：

A. 加密传输（解决对称加密的密钥分发问题）

接收方Bob生成一对密钥，并把他的公钥公开发布（就像公开他的信箱地址和投递口）。

发送方Alice想给Bob发密信，就用Bob的公钥加密信息。

加密后的密文只有用Bob的私钥才能解开。由于私钥只有Bob自己有，所以即使密文被截获，别人也无法解密。

B. 数字签名（验证身份和防篡改）

发送方Alice想证明这封信确实是她发的且没被改动，就用她自己的私钥对文件生成一个“数字签名”。

她把文件和签名一起发出。

接收方用Alice公开的公钥去验证这个签名。如果验证通过，就证明：①这文件确实来自Alice（身份认证）；②文

件在传输过程中没被篡改（完整性）。

虚拟币定义

虚拟币是基于区块链技术的去中心化数字资产，以比特币（BTC）为代表，通过密码学确保交易安全和控制供应。

虚拟币分类

公链币

定义**：**一条独立区块链（公有链）的原生加密货币，是这条链的“血液”和“燃料”。

典型例子：比特币（BTC）、以太币（ETH）、Solana（SOL）、Avalanche（AVAX）

链上代币

定义：在已有公链上，通过智能合约标准（如以太坊的ERC-20）创建的代币。它没有自己独立的区块链

典型例子：Uniswap（UNI，治理代币）、Chainlink（LINK，功能代币）、USDT（稳定币）、各种Memecoin（如狗狗币的仿盘）。

空气币

定义：这是一个贬义的民间说法，特指那些没有任何实际产品、技术、应用场景或团队背景，纯粹为了炒作和割韭菜而发行的代币。

虚拟货币分发币

定义：通常指中心化机构（如交易所、内容平台）发行的，用于其自身生态系统内激励和消费的代币。（一般称为“平台积分”或“生态激励代币”）

典型例子：币安币（BNB）、OKB、火币积分（HT）等交易所平台币。注意，像BNB早期是ERC-20代币，后来发展出了自己的公链（BSC），因此BNB现在也具有了公链币的属性。

清除浮动

  • 场景：浮动元素会脱标，如果父级没有高度，子级无法撑开父级高度（可能导致页面布局错乱）

  • 解决方法：清除浮动带来的影响

    • 额外标签法

      • 在父级元素的最后添加一个块级元素，设置CSS属性 clear:both(元素类名习惯设置为clearfix)

    • 单伪元素法

      • 给父级元素增添一个伪元素类名

    • 双伪元素法（推荐）

      • 给父级元素前后都添加伪元素设置为表格（table）显示模式

      • 注意：before伪元素解决外边距塌陷问题，after清除浮动

    • overflow

      • 父元素添加CSS属性 overflow:hidden

  • 注意：如果标签自己有样式，则生效自己的样式，不会继承父级样式

层叠性

  • 相同的属性会产生覆盖：后面的CSS属性覆盖前面的CSS属性
  • 不同的属性会产生叠加：不同的CSS属性都生效

优先级

  • 优先级也叫权重，当一个标签用了多个选择器时，基于不同种类的选择器的匹配规则
  • 规则：优先级高的选择器样式优先生效
  • 公式：通配符选择器<标签选择器<类选择器<id选择器<行内样式<!important
  • ！important：提权功能，提高权重，优先级到最高，慎用

优先级-叠加计算规则

  • 叠加计算：如果是复合选择器，需要进行权重叠加计算
  • 公式：行内样式，id选择器个数，类选择器个数，标签选择器个数（每一级之间不存在进位）
  • 规则：
    • 从左到右依次比较选个数，同一级个数多的优先级高，如果个数相同，则向后比较
    • !important权重最高
    • 继承权重最低

合并单元格

合并单元格的步骤

  1. 明确合并的目标
  2. 保留最左最上的单元格，添加属性（取值是数字，表示需要合并的单元格数量）
    1. 跨行合并，保留最上单元格，添加属性rowspan
    2. 跨列合并，保留最左单元格，添加属性colspan
  3. 删除其他赘余单元格（倘若不删除赘余单元格，会出现单元格扩大导致表格混乱）

注意：合并单元格必须在自己的结构体内合并，比如表格头部内容不能和主题内容进行合并
